# Contributing

## Delivery guardrails

1. **Do not push without explicit user/maintainer consent.**
2. **Always validate in a local browser first** for UI/interaction changes before requesting a push.
3. **Keep code minimal and human-readable.** This is a demo, not a production system.

## Local validation checklist

1. Start app locally.
2. Open browser and verify room creation, chat send, and panel rendering.
3. Verify invite/share actions if changed.
4. Run local build/tests before preparing commit.
